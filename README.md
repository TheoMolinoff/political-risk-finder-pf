# political-risk-finder-PF

A deterministic **political & regulatory risk** assessment tool for renewable-energy
**project finance** (utility-scale wind, solar, and battery storage) across US states.

Select a state and a technology, and the tool returns a tiered risk assessment —
**federal / state / local** — differentiated by technology and mapped to where each
risk lands in a project's financing (tax equity, debt sizing, construction, offtake/merchant).

> **What this is, and isn't.** This is a *projection and illustration* instrument, not a
> predictive model. Its credibility comes from **traceable inputs and a transparent,
> disclosed methodology** — every score is computed deterministically from structured,
> sourced data, never generated by a language model. Scores are not backtested outcome
> predictions; they are a structured, auditable read of documented political/regulatory risk.

---

## How it works

Risk **scores are deterministic** — produced by a transparent function over structured
factor data ([`src/lib/scoring.ts`](src/lib/scoring.ts)). For a given state + technology:

1. Each risk **factor** carries a 0–3 severity per technology, a tier (federal/state/local),
   and a capital-impact channel.
2. Each tier score is the normalized sum of its factor severities.
3. **Mitigants** (e.g. a binding state renewable mandate) are tier-scoped reductions —
   a state policy can offset state-tier risk but never federal or local risk.
4. The overall score is a per-technology weighted blend of the three tiers, mapped to a
   level: **High / Elevated / Moderate / Low**.

A language model is used only to write prose explanations of pre-computed scores — it
never invents a score, a law, a date, or a number.

## Data anchoring

Where hard data exists, severities are **derived from public datasets by committed code**,
not hand-keyed — so they are reproducible and auditable:

- **Local siting risk** is computed from the **Sabin Center Opposition Report** dataset
  (counts of local restrictions and contested projects per state) via
  [`scripts/derive-siting.ts`](scripts/derive-siting.ts), which applies a disclosed rubric
  and writes [`src/data/siting-severities.json`](src/data/siting-severities.json).
- **Contested-project records** (real named projects, counties, and status) are extracted via
  [`scripts/derive-contested.ts`](scripts/derive-contested.ts) into
  [`src/data/contested-projects.json`](src/data/contested-projects.json).

Factors without a clean dataset (e.g. forward-looking legislative or policy-uncertainty
risks) are labeled as **qualitative judgment** rather than dressed up as quantitative —
honest labeling over false precision.

Full methodology, rubrics, and known limitations are documented in
[`METHODOLOGY.md`](METHODOLOGY.md).

## States covered

Eight states spanning every major US market structure:

| State | Market structure |
|-------|------------------|
| Texas | ERCOT energy-only merchant |
| Virginia | PJM capacity market + VCEA mandate |
| California | CAISO (curtailment) + SB 100 / AB 205 |
| Utah | Regulated, hostile utility (IRP plans no new renewables) |
| Mississippi | Regulated SE, discretionary utility procurement |
| Arizona | Regulated, favorable procurement, active county solar moratoria |
| Nevada | Regulated, binding 50%-by-2030 RPS |
| Illinois | Restructured + CEJA/CRGA mandate, mandate-to-deployment gap |

## Tech stack

Next.js (App Router) · TypeScript · Tailwind CSS · pnpm. No database — risk data lives in
typed/structured files. Data-derivation scripts run at build time (`node scripts/derive-*.ts`).

## Running locally

```bash
pnpm install
pnpm dev
# open http://localhost:3000
```

To regenerate the data-anchored siting severities or contested-project data after a new
Sabin dataset is dropped into `data/sabin/`:

```bash
node scripts/derive-siting.ts
node scripts/derive-contested.ts
```

## Data sources

- **Sabin Center Opposition Report** — local siting restrictions & contested projects
- **EIA / ISO public data** — generation mix, capture prices, capacity auctions
- **DSIRE** — state RPS / incentive policy
- **Utility IRP filings** — procurement posture in regulated markets
- Public reporting, cited per factor

## Status & roadmap

Active, in development. Built complete across eight states; refining toward deeper
data-anchoring across all factor types, confidence ranges in place of point scores, a
per-state contested-project map, and a financial-impact layer.

## Disclaimer

This tool is for research and illustration. It is not investment, legal, or financial
advice, and its scores are not guarantees or predictions of project outcomes.
